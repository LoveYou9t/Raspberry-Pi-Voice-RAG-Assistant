# 服务启动文档（简版）

本文档用于快速启动当前语音 RAG MVP 服务。

## 1. 启动前准备

1. 已安装 Docker Desktop，并确保 Docker 正常运行。
2. 当前目录为项目根目录。
3. 如需调整模型或参数，可先修改 .env 文件。

## 2. 首次启动（推荐）

1. 在项目根目录执行：
   docker compose up -d --build
2. 查看服务状态：
   docker compose ps
3. 看到以下容器为 Up 状态即可：
   edge_ollama、edge_fastapi、edge_frontend

## 3. 访问地址

1. 前端页面：<http://localhost:8080>
2. 后端健康检查：<http://localhost:8000/healthz>

## 4. 常用运维命令

1. 查看实时日志：
   docker compose logs -f
2. 重启全部服务：
   docker compose restart
3. 停止并删除容器（保留数据卷目录）：
   docker compose down

## 5. 模型预热（可选）

如果首次启动后模型未准备好，可手动执行：

1. 启动 Ollama：
   docker compose up -d ollama_server
2. 拉取模型：
   docker exec edge_ollama ollama pull llama3.2:1b
3. 启动后端与前端：
   docker compose up -d --build fastapi_backend frontend_client

## 6. 简单排查

1. 前端打不开：确认 edge_frontend 容器为 Up，并检查 8080 端口未被占用。
2. 后端无响应：确认 edge_fastapi 为 Up，执行 docker compose logs -f fastapi_backend 查看错误。
3. LLM 无返回：确认 edge_ollama 为 Up，并执行 docker exec edge_ollama ollama list 检查模型是否存在。
4. 无语音播放：先看浏览器是否允许麦克风权限，再检查后端日志中的 WebSocket 连接信息。

## 7. 目录提示

1. 知识库目录：knowledge_base
2. 向量数据目录：lancedb_data
3. 模型数据目录：ollama_data

将新的 .md 或 .txt 放入 knowledge_base 后，可在后端容器内运行知识同步脚本更新索引。
